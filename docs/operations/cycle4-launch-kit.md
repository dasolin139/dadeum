# Cycle 4 — 다듬(Dadeum) 上线即发射 Launch Kit

> 运营总监（Paul Graham 思维模型）· 目标：人类点部署的 5 分钟后，靠"创始人亲手请人试用"把 smoke test 拿到有意义的付费信号。
> 阶段判断：**pre-PMF / pre-launch**。瓶颈不是产品，是分发。这份 kit 是执行级，复制粘贴就能发。

## 0. 合规红线（每一段文案都必须过这一关）

绝对禁止出现：`detector` / `bypass` / `탐지` / `우회` / `AI 검사 통과` / `GPT 감지 회피` 及任何同义暗示。
只能说：**윤문 · 글 품질 개선 · 번역투 제거 · 자연스러운 한국어 · 사람이 쓴 것처럼 읽히게**。
自检口诀：如果一句话把卖点建立在"骗过某个检测系统"上，删掉重写。卖点永远是"读起来更像人写的好文章"。

买家画像回顾：**번역/현지화 종사자 · 브랜드 뉴스레터 운영자 · 이커머스 상세페이지 작성자**。共同点——每一篇都押上自己的声誉/转化率，AI 味 = 掉价，所以 WTP 高。分发要去他们**已经聚集、已经在抱怨"AI 문체"的地方**，一个一个手动请人试。

## 3. Product Hunt 발射 물료

**Tagline (영문, ≤60자)**
`Make AI-written Korean read like a human wrote it` (48자)
대안: `Polish AI Korean into natural, human-sounding text` (50자)

**한국어 한 줄**
`AI로 쓴 한국어를, 내용은 그대로 두고 문체만 사람이 쓴 것처럼 다듬어 드려요.`

**First comment (창업자 스토리)**
```
Hi PH 👋 I'm the maker of 다듬 (Dadeum).

I write a newsletter and product copy in Korean, often starting from an AI draft.
The facts were always fine — but the *style* gave it away: translationese, every
sentence in the same rhythm, endless "~할 수 있습니다". Readers can feel it.

So I built a tool that fixes only that. It keeps every fact, number, quote, and
proper noun untouched, and rewrites just the tone, rhythm, and phrasing into
natural-sounding Korean. Paste → one click → see original vs. polished side by
side → copy.

It's for people whose reputation rides on each piece of text — translators,
newsletter writers, e-commerce copywriters. No login, 3 free polishes a day.

I'd love feedback from Korean speakers especially: paste something and tell me
where it still reads unnatural — I'll fix it fast. Thank you for trying it 🙏
```

## 5. 成功/失败 硬指标 (kill switch 정렬)

측정 안 되는 발射은 무효다. 최소한 **방문 → 실사용(다듬기 실행) → 유료 CTA 클릭** 3단 퍼널을 웹훅/분석으로 잡아라.

### 채널 단위 판정 (게시 후 72시간)
| 신호 | 유효 (계속·확장) | 애매 (1회 더 시도) | 무효 (교체) |
|---|---|---|---|
| 방문 | ≥ 30 | 15–29 | < 15 |
| 실사용(도구 실행) | ≥ 10 | 4–9 | < 4 |
| 유료 CTA 클릭 | ≥ 1 | 0 (방문은 있음) | 0 (방문도 없음) |

- **무효 채널은 미련 없이 버리고 리스트의 다음 채널로.** 시간은 유한하다.
- 유효 채널은 2차 포스트·후기 인용으로 화력 재투입.

### 1주차 전체 목표 (smoke test "신호 있음"의 최소선)
- 총 방문 **200–400**
- 실사용 세션 **60–100** (방문→실사용 전환 ≥ 25% = 랜딩/가치 전달 OK)
- **free→paid 클릭 의향 ≥ 2%** (실사용 대비 유료 CTA 클릭)
- 자발적 재방문/재사용 흔적 ≥ 5건 (얇지만 留存 신호)

### Kill switch (이미 확정, 재확인)
- **4주차 free→paid 클릭 < 2%** → 관停
- **付费(의향) 留存 < 50%** → 관停
- 위 둘 중 하나라도 터지면 이 제품에 더 시간 쓰지 않는다.

### 운영 陷阱 경고 (虚荣지표 금지)
- "노출 수 / 좋아요 / 조회수"는 虚荣지표다. **실사용 세션과 유료 CTA 클릭만** 대시보드에 올려라.
- 방문만 많고 실사용 전환 < 15%면 채널 문제가 아니라 **랜딩/가치 전달 문제** — 카피를 고쳐라.
- 실사용은 많은데 유료 클릭 0이면 **WTP 없음 신호** — 4주 기다리지 말고 조기 경고로 취급.
- 절대 과·조기 확장 금지: 1주차는 3~5개 채널 손으로만. 자동화·유료광고는 신호 확인 후.

## 요약 (내부 메모)

- **top 채널**: 1) 디스콰이엇(가장 안전·첫 실사용자) 2) 번역가 커뮤니티(최고 WTP) 3) 뉴스레터 씬 4) 이커머스 셀러 카페 5) Product Hunt.
- **가장 먼저**: 배포 직후 계측 3종(실사용·유료클릭·웹훅) 점검 → Day1에 **디스콰이엇 메이커 로그 + 긱뉴스 + Threads** 동시 발射, 댓글은 창업자가 직접 손으로 응대.
- **신호 가능성**: 이 조합이면 1주차 200–400 방문·60–100 실사용은 현실적으로 도달 가능 → free→paid 2% 임계를 **찍을 수도, 못 찍을 수도** 있는 "진짜 시험대"에 오른다. 즉 **의미 있는 신호를 뽑기엔 충분하다.** 단 전제 조건 하나 — 배포 시 유료 CTA 클릭과 실사용이 **실제로 계측**돼야 한다. 계측이 없으면 아무리 잘 발射해도 smoke test는 죽는다. 그러니 T+5분의 첫 액션은 홍보가 아니라 **계측 확인**이다.

### 6-3. 발射 순서 (배포 전 vs 후)
